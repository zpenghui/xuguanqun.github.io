JavaScript中的“Top Level Await”怎么用（译文）

  • 译文
作者: 徐冠群

目录

  1. 1. 模块加载失败时使用回调
  2. 2. 使用加载速度最快的资源
  3. 3. 初始化资源

  5. 5. 控制台中，在异步函数之外使用await

正文

以前，为了使用 await ，代码必须位于标记为异步的函数中。这意味着您不能在任何函数符号之外使用 await 。 Top Level Await 可以使模块像异步函数一样工作。

模块是异步的，并且具有导入和导出的功能，并且这些模块也在顶层表示。实际的含义是，如果您想提供一个依赖于某些异步任务的模块来执行某项操作，那么您实际上没有好的选择

Top Level Await 可以解决此问题，并使开发人员可以在异步函数之外使用 await 关键字。使用 Top Level Await ，ECMAScript 模块可以等待资源，从而使其他导入资源的模块在开始评估其主体之前等待，而且如果模块加载失败或用作加载第一个模块，也可以将加载好的依赖项回退资源下载。

注意：

  • 顶层等待仅在顶层模块上起作用。不支持经典脚本或非异步功能。
  • 截至撰写本文时（2020年2月23日）ECMAScript 为第三阶段。

在顶层等待的情况下，下一个代码将按您期望的方式在模块中工作

# 1. 模块加载失败时使用回调

以下示例尝试从 first.com 加载 JavaScript 模块，如果失败则使用回调：

//module.mjs
let module;
try {
  module= await import('https://first.com/libs.com/module1');
} catch {
  module= await import('https://second.com/libs/module1');
}

# 2. 使用加载速度最快的资源

此处 res 变量通过首先完成的下载来初始化

//module.mjs
const resPromises = [    
    donwloadFromResource1Site,
    donwloadFromResource2Site
 ];
const res = await Promise.any(resPromises);

# 3. 初始化资源

顶层等待允许你可以在模块中等待 promises ，就像将它们包装在异步函数中一样。例如，初始化应用程序：

//module.mjs
import { dbConnector} from './dbUtils.js'
//connect() return a promise.
const connection = await dbConnector.connect();
export default function(){connection.list()}

# 4. 动态加载模块

这允许模块使用运行时值来确定依赖性。

//module.mjs
const params = new URLSearchParams(window.location.search);
const lang = params.get('lang');
const messages = await import(`./messages-${lang}.mjs`);

# 5. 控制台中，在异步函数之外使用await

在使用 async / await 之前，尝试在 async 函数之外使用 await 会导致: “ SyntaxError: await is only valid in async function” Now, you can use it without it being inside in an async function.

这已在 chrome 80 和 firefox 72.0.2 DevTools 中进行了测试。但是，此功能是非标准功能，无法在 nodejs 中使用。

const helloPromise = new Promise((resolve)=>{
  setTimeout(()=> resolve('Hello world!'), 5000);
})
const result =  await helloPromise;console.log(result);
//5 seconds later...:
//Hello world!

# Implementations

  • V8 with flag: — — harmony-top-level-await
  • Webpack (experimental support in 5.0.0)
  • Parser support has been added to Babel (babel/plugin-syntax-top-level-await).
